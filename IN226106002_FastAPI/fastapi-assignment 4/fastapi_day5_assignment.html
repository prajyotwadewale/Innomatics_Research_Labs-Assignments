Scenario: The customer sees the USB Hub on the website and tries to add it to the cart. But the USB Hub (id=3) is out of stock. Your API must reject it with the correct error.
Your Tasks
🧪

Try POST /cart/add?product_id=3 — USB Hub is out of stock

❌

Must return 400 Bad Request with an out-of-stock error message

✅

GET /cart after — USB Hub must NOT be in the cart. Cart still has 2 items.

🧪

Also try POST /cart/add?product_id=99 — must return 404 Not Found

Test These Error Cases
POST/cart/add?product_id=3→ 400 Bad Request — out of stock
POST/cart/add?product_id=99→ 404 Not Found
GET/cart→ still 2 items only, total still 1097
Expected Error — 400 Bad Request
{ "detail": "USB Hub is out of stock" } // Status line: 400 Bad Request
💡 Show Hint
Hint — Out-of-stock check in add_to_cart()
# This check should already be in your add_to_cart() function: if not product["in_stock"]: raise HTTPException( status_code=400, detail=f"{product['name']} is out of stock" ) # 400 = Bad Request — the product exists but can't be ordered # 404 = Not Found — the product_id doesn't exist at all
